# Multi-Modal Food Discovery System

**RAG application · Open Food Facts dataset · 15-week ML project**

A multi-modal search and retrieval system for food products built on the
Open Food Facts dataset. Users can find products using natural language,
product images, or nutritional filters — with conversational answers
generated by a local LLM.

---

## System Architecture

```
User query (text / image)
        │
        ▼
 Query Rewriter (Qwen2.5)        ← detects follow-up vs. new topic
        │
        ▼
  FAISS Vector Search
  ├── Text index  (sentence-transformers/all-MiniLM-L6-v2)
  └── Image index (google/vit-base-patch16-224-in21k)
        │
        ▼
  RRF Fusion (c=60)              ← combines text + image scores
        │
        ▼
  Safety Guardrails              ← pre-generation dietary conflict check
        │
        ▼
  RAG Answer (Qwen2.5-1.5B)     ← context-aware nutritionist response
        │
        ▼
  Streamlit Chat UI
```

---

## Features

**Multi-modal retrieval:**
- Text search via sentence-transformers embeddings + FAISS
- Image search via ViT (Vision Transformer) embeddings + FAISS
- Hybrid mode combining both with **Reciprocal Rank Fusion (RRF)**

**Conversational RAG:**
- LLM-based query rewriting — distinguishes follow-up questions from new topics, adds missing context automatically
- Local generative LLM (Qwen/Qwen2.5-1.5B-Instruct) for nutritionist-style answers
- Full conversation history maintained across turns

**Safety guardrails:**
- Pre-generation dietary conflict detection (vegan, vegetarian, gluten-free, peanut-free)
- Ingredient scanning before LLM call — conflicts surfaced as explicit warnings

## Setup

```bash
# 1. Clone the repository
git clone https://github.com/kamilb222/food-discovery-rag.git
cd food-discovery-rag

# 2. Create virtual environment
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Configure environment
cp .env.template .env
# Edit .env and add your DATA_DRIVE_URL to download the dataset
```

## Usage

**Step 1** — Run notebooks in order to prepare data and build FAISS indices:
```
notebooks/01_Data_Ingestion_Cleaning_EDA.ipynb
notebooks/02_base_NLP_CV.ipynb
notebooks/03_Advanced_Embeddings_ViT_BERT.ipynb  ← builds FAISS indices
notebooks/04_RAG_Pipeline.ipynb                  ← RAG pipeline demo
```

**Step 2** — Launch the Streamlit app:
```bash
streamlit run app/app.py
```

The app will be available at `http://localhost:8501`.
